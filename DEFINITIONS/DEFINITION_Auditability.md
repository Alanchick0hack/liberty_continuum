# DEFINITION — Auditability

## Definition
Auditability is the property that decisions can be inspected after the fact with enough traceability to verify inputs, constraints, outputs, and accountability.

## Required Properties
- Logged decision record
- Versioned system identity
- Input sources enumerated
- Constraint/policy reference
- Appeal route

## Not This
- “Trust us” black boxes
- Unlogged automation
- Non-reproducible outcomes
